# Test Driven Documentation

## 목적
- 요청자와의 간극을 없애고자 도입
- 요구사항에 대한 구현 가능 여부를 신속히 파악 _*근거 보충 필요_
- 이슈 및 의문에 대한 취합이 비교적 빠르게 이루어짐 _*근거 보충 필요_


## Cycle
1. 기획 문서를 본다.
2. 요구 사항을 파악한다.
3. 요구 사항에 대한 테스트를 작성한다.
4. 중복되는 테스트를 제거한다.
5. 테스트에 맞는 작업을 정의한다.
